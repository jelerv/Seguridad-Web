# Módulo 4: A02:2025 – Security Misconfiguration

**Objetivos de aprendizaje:**
- Entender qué es Security Misconfiguration y por qué subió al puesto #2 en OWASP Top 10:2025.
- Identificar las configuraciones inseguras más comunes en servidores web, frameworks y entornos cloud.
- Detectar problemas típicos: CORS mal configurado, headers de seguridad faltantes, mensajes de error verbosos, directorios expuestos.
- Aplicar hardening en aplicaciones Flask y entornos modernos.
- Configurar correctamente headers de seguridad (HSTS, CSP, X-Frame-Options, etc.).
- Diferenciar misconfiguraciones en aplicaciones web vs APIs.

**Duración recomendada:** 5 horas (1.5 horas teoría + 3.5 horas labs).

**Posición en OWASP Top 10:2025:** #2
