# Módulo 5: A03:2025 – Software Supply Chain Failures

**Objetivos de aprendizaje:**
- Entender la evolución de "Vulnerable and Outdated Components" (A06:2021) a **Software Supply Chain Failures** (A03:2025).
- Identificar riesgos en dependencias, pipelines CI/CD, repositorios de paquetes y build processes.
- Conocer ataques reales de supply chain (SolarWinds, Log4Shell, XZ Utils, Bybit 2025, etc.).
- Generar y usar Software Bill of Materials (SBOM).
- Aplicar herramientas de análisis de composición de software (SCA): OWASP Dependency-Check, pip-audit, npm audit, Trivy, Snyk.
- Implementar buenas prácticas: version pinning, checksum verification, trusted sources y SBOM.

**Duración recomendada:** 5 horas (1.5 horas teoría + 3.5 horas labs).

**Posición en OWASP Top 10:2025:** #3

**CWEs asociadas (principales):**
- CWE-1395: Dependency on Vulnerable Third-Party Component
- CWE-1104: Use of Unmaintained Third Party Components
- CWE-1329: Reliance on Component That is Not Updateable

**Referencias oficiales:**
- OWASP Top 10 2025 – A03: Software Supply Chain Failures
- OWASP Software Component Verification Standard (SCVS)
- CycloneDX y SPDX (formatos de SBOM)
