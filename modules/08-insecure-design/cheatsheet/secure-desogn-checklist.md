# Checklist de Secure Design

- [ ] ¿Se realizó Threat Modeling (STRIDE) en la fase de diseño?
- [ ] ¿Los flujos de negocio críticos tienen controles de autorización?
- [ ] ¿Se definen límites de negocio (rate limiting, monto máximo, etc.)?
- [ ] ¿Los datos sensibles están protegidos desde el diseño?
- [ ] ¿Se considera el principio de menor privilegio?
- [ ] ¿Los errores revelan información sensible?
- [ ] ¿Se diseñó pensando en ataques (fail-safe defaults)?
- [ ] ¿Se incluye logging y monitoring desde el diseño?
- [ ] ¿Se planificó manejo seguro de excepciones?
